## 三、架构设计

### 3.1 进程架构

```
┌─────────────────────────────────────────────────────┐
│  Electron Main Process (electron/main.ts)           │
│  - BrowserWindow 管理 (frame: false, 无边框窗口)     │
│  - IPC Handlers: 文件读写、数据存储、封面管理、导入导出  │
│  - 数据持久化: reader-data.json (userData 目录)       │
│  - 封面存储: userData/covers/ 目录                    │
└────────────────────┬────────────────────────────────┘
                     │ IPC (contextBridge)
┌────────────────────┴────────────────────────────────┐
│  Renderer Process (src/)                            │
│  - React 19 + Tailwind CSS 4                        │
│  - 状态管理: 自定义 Hook (useReaderStore)             │
│  - 三种阅读器: epubjs / react-pdf / 原生渲染           │
└─────────────────────────────────────────────────────┘
```

### 3.2 数据流

```
用户操作 → App.tsx → useReaderStore (React State)
                          ↓
                    window.electronAPI.store.set()
                          ↓
                    IPC → main.ts → fs.writeFileSync (reader-data.json)
```

**关键设计决策:**
- **无 Redux/Zustand**: 使用 React 原生 `useState` + 自定义 Hook 管理状态
- **防抖保存**: `debouncedSave` 3 秒延迟写入磁盘，避免频繁 I/O
- **即时保存**: 关键操作 (`saveData`) 直接写入，不经过防抖
- **乐观更新**: UI 先更新，异步写入存储

### 3.3 安全设计

- `contextIsolation: true` - 渲染进程与 Node.js 隔离
- `nodeIntegration: false` - 禁止渲染进程直接访问 Node API
- 所有文件操作通过 IPC Handler 桥接
- `will-navigate` 事件拦截，阻止页面跳转

## 四、核心模块详解

### 4.1 状态管理 - `useReaderStore.ts`

这是整个应用的核心，237 行代码管理了所有业务逻辑：

| 函数 | 职责 |
|------|------|
| `loadData` | 启动时从 electron-store 加载数据 |
| `saveData` | 同步写入 (关键操作) |
| `debouncedSave` | 3 秒防抖写入 (阅读进度) |
| `importBooks` | 文件对话框导入 |
| `importBooksByPaths` | 拖放路径导入 |
| `addBooks` | 去重 + 创建 Book 对象 + 异步提取封面 |
| `openBook` | 设置活跃书籍 + 记录会话开始时间 |
| `closeBook` | 计算阅读时长 → 生成 ReadingSession → 保存 |
| `updateBookProgress` | 更新进度 (防抖) |
| `toggleFavorite` | 切换收藏 |
| `removeBook` | 删除书籍及关联会话 |

**封面提取流程:**
```
导入书籍 → 立即显示随机渐变色封面
         → 异步提取真实封面 (epubjs/pdfjs)
         → 保存到 userData/covers/
         → 更新 Book.coverPath → 触发重渲染
```

### 4.2 EPUB 阅读器 - `EpubReader.tsx`

最复杂的组件 (326 行)，功能包括：

- **两种阅读模式**: 翻页 (paginated) / 滚动 (scrolled-doc)，通过 localStorage 持久化
- **目录导航**: 从 epubjs 加载 TOC，支持多级目录
- **进度记录**: 基于 CFI (Canonical Fragment Identifier) 精确定位
- **位置生成**: 后台 `locations.generate(100)` 实现百分比进度
- **键盘/滚轮导航**: 翻页模式支持 ← → PageUp/Down/Space
- **点击区域**: 左 1/4 上一页，右 1/4 下一页
- **章节注入**: 滚动模式下自动注入"下一章"按钮
- **字体调整**: 14px ~ 32px

**技术要点:**
- 动态 `import('epubjs')` 避免 React Hook 冲突
- `requestAnimationFrame` 等待容器尺寸就绪
- `progressRef` ref 闭包避免 stale closure
- 完善的 cleanup (destroy rendition + book)

### 4.3 PDF 阅读器 - `PdfReader.tsx`

基于 `react-pdf` (pdfjs 封装)：

- 读取文件 buffer → `pdfjs.getDocument` 渲染
- 支持缩放 (0.5x ~ 3x，步进 0.2)
- 页码跳转 + 上下页导航
- 进度基于当前页码/总页数
- `progressData` 存储当前页码字符串

### 4.4 TXT 阅读器 - `TxtReader.tsx`

最轻量的阅读器：

- 读取 UTF-8 文本 → 按换行符分段渲染
- 滚动位置记录: `scrollTop / maxScroll * 100`
- 恢复滚动位置: 读取 `readingProgress` 百分比还原
- 字体调整: 14px ~ 32px
- 键盘导航: PageUp/Down, Home/End

### 4.5 封面提取 - `coverExtractor.ts`

- **EPUB**: epubjs 解压 → `book.coverUrl()` → fetch blob → FileReader → dataURL
- **PDF**: pdfjs 渲染第 1 页到 Canvas → `canvas.toDataURL('jpeg', 0.75)`
- 封面宽度统一缩放到 400px

### 4.6 数据同步 - `SyncView.tsx`

三个功能：
1. **导入阅读数据**: 选择 JSON 文件 → 覆盖 reader-data.json → 重载
2. **导出阅读数据**: 复制 reader-data.json 到指定位置
3. **导出书籍文件**: 批量复制原始文件到指定目录

### 4.7 统计视图 - `StatsView.tsx`

- 4 个统计卡片: 累计阅读、已读书籍、书架藏书、日均阅读
- 近 7 天柱状图 (纯 CSS 实现)
- 阅读排行 (按累计阅读时间排序)
- 最近 10 条阅读记录

## 五、UI/UX 设计特点

1. **无边框窗口**: `frame: false` + 自定义标题栏 + `drag-region`
2. **极简配色**: 以 zinc 色系为主，黑白灰为主调
3. **可展开侧边栏**: hover 展开 (72px → 240px)，显示更多文字信息
4. **响应式网格**: 书架 `grid-cols-2` 到 `grid-cols-6` 自适应
5. **拖放导入**: 支持文件拖放到窗口任意位置
6. **右键删除**: 书籍卡片右键弹出确认对话框
7. **加载动画**: 统一的旋转圆环 loading
8. **微交互**: hover 阴影、缩放、渐变过渡
